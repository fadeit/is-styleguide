OLS Front-end Styleguide

    Before you start

    What's inside

    This styleguide uses a fork of Bootstrap from here, without the components that require JavaScript.
    The purpose of this thing is to achieve a more consistent front-end design and keep everybody in the team sane.
    It's also meant to cover all (or most) situations that you'll encounter during development, therefore you should never have to create your own element/style. Use the code samples and be happy!

    Don't forget, you can resize this page to see how the styles adapt to different viewport sizes!

    Colors

    The colors in this list are the only colors to be used.

    Conventions

    • Color variables are prefixed with col.
    • Darker, contrasting versions of a color are sufixed with _c: @w (white) and @w_c (white contrast)

    Scaffolding: layout & grid system

    The grid system is made out of 24 columns and works exactly as the Bootstrap Grid, times 2. The grid double-up is chosen for flexiblity. In markup, it just means doubling up all the CSS column sizes.

    Breakpoints

    There are 4 breakpoints. Properties that do not apply to a breakpoint are available up to a resolution of 767 pixels.

    LESS variables names: when using (min-height) with media queries, the name of the LESS variable is @bp_[breakpoint_size]. When using (max-height) with media queries, the name of the LESS variable is @bp_[breakpoint_size]_max.
    This prevents overlapping when using both min and max height.

    It's a good idea to stick with min-height (that's usually a consequence of mobile-first design).

    LESS name (min-height) LESS name (max-height) When it applies (pixels)
    @bp_sm @bp_sm_max > 767px
    @bp_md @bp_md_max > 991px
    @bp_lg @bp_lg_max > 1199px
    @bp_xl @bp_xl_max > 1599px

    As with Bootstrap, column sizes can be specified in the HTML markup for each breakpoint. A class of col-md-6 will have the size of 6 columns on viewports bigger than 991 pixels. To get an adaptive layout, column sizes can be stacked on one element.
    For example, a class of col-xs-24 col-md-6 will have the size of 24 columns up to 991 pixels and 6 columns for viewports bigger than that.

    The table transformation will be applied according to the specified breakpoint (similar to how columns work). In this example that is @bp_md or viewport sizes bigger than 991 pixels.

    Element @bp_sm (> 767px) @bp_md (> 991px) @bp_lg (> 1199px) @bp_xl (> 1599px)
    Container .container-height-sm .container-height-md .container-height-lg .container-height-xl
    Row .row-height-sm .row-height-md .row-height-lg .row-height-xl
    Column .col-height-sm .col-height-md .col-height-lg .col-height-xl

    The [element]-height-[breakpoint] class should be added at the end of the element to avoid conflicts.
    For example, class="row-fluid col-xs-24 col-md-4 col-height-md" is the correct way of applying the equal column height class.

    Buttons & Links

    Primary button

    This is the 'normal' button. It supports a 'disabled' state (also used for loading buttons). A 'dark' button is available for primary actions with lower priority, such as sending a message. The dark button can be used by appending the dark class to the btn btn-primary class.

    Secondary buttons

    Buttons to be used in other circumstances, such as 'important' (danger). Disabled states are also available for them.

    Small buttons

    Adding a btn-sm class to any button will decrease the button size.

    Round buttons

    These buttons should be used for 'form steps'. Their main function is to present an overview of the steps, as well as the current step position. This button supports 3 states:

    • (default - no class): green color & supports hover
    • (disabled - btn-disabled): gray color, transparent, no hover
    • (active - btn-active): dark green color, no hover, no pointer

    To center a rounded button group, wrap the buttons elements with a rounded-button-group parent.
